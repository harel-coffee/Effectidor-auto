What is Effectidor's output?


Effectidor produces both tabular and graphical outputs. In the tabular output there are:

  1. The T3Es' predictions, i.e., each ORF gets a score that reflects its likelihood to encode a type 3 effector, as produced by the machine-learning. In this file you will have the locus_tags sorted by the ORFs' likelihood to encode effectors. The protein sequence will appear next to it, in addition to the protein annotation, if was available in the input data.
    The 10 best scoring, unlabeled predictions, as well as the positive set that was used to train the classifier, will be displayed on the screen, while the predictions for all the ORFs in the genome (labeled data as well as unlabeled data) will be available in a downloadable Excel file.
  2. Pseudogenes file - an Excel file with all the predictions of ORFs that are annotated as pseudogenes.
    Highly scored samples available in this file are potentially with effector origin that was lost in evolution.
    This file will be empty if the input ORFs file did not contain annotations.
  3. Feature importance table. This table shows the features by their contribution to the learning, and their relative importnce. It is available as a downloadable csv file.
  4. Raw features file. The features file, containing all the features that were extracted from the input data, that was used to train the machine learning algorithm, is also available as a downloadable csv file.

The graphical output shows analysis of the features. The 10 most contributing features will be compared between the two classes - effectors and non-effectors, in violin plots.

You can see an example for the output in the running example here.


How do I interpret the results?


The main results, the predictions file, contains the ORFs sorted by their likelihood to encode T3Es.
To identify candidates to be novel T3Es, look at the top rows of the table, and search ORFs with a high score.
If you used an input Effectors file, look for samples that are not already known to be effectors. These samples will have a question mark (?) in the column "is_effector".
If you have not used an input Effectors file, The samples marked with "yes" in the "is_effector" column are homologous to known T3Es, and the highly scored samples marked with a question mark (?) are potentially novel T3Es.
For your convenience, the highly scored samples are highlighted with dark green that becomes lighter as the score decreases.
If no candidate is found at the top of the file, you can carry on and check lower rows in this file.
For your convenience, the amino acid sequence will be available to each ORF, as well as the annotation (if existed in the input file).

In general, an ORF with a high score and an ambiguous annotation is a good candidate to be a T3E.


How is Effectidor evaluated and where can I see its predicting accuracy?


In the learning process Effectidor randomly leaves 20% of the labeled data (effectors and non-effectors) aside, as a test set. Each learning algorithm is trained on the remaining 80% training data, using cross validation, and in the end it is evaluated on the untouched 20% test set.
The measurement used to evaluate Effectidor's accuracy is the Area Under the Precision Recall Curve (AUPRC). The closer it is to 1, the more accurate the classifier is. You can see the resulting AUPRC achieved on the test set, in the downloadable predictions file.


What input does Effectidor require and what do they use for?


Effectidor has one obligatory input - ORFs file.
This file is a FASTA file with all the ORFs in the genome. See instructions on how to download this file here.
Some of the ORFs in this file (effectors and non-effectors) will be used to train the machine-learning algorithms, and based on the trained classifier, the main output - prediction for each ORF - will be performed.
In addition, it is recommended to supply an Effectors file. This file should contain all the known effectors' rocords, as appear in the ORFs file. If you do not supply such a file, a homology search against an internal effectors dataset will be performed to constitute the "known effectors" ORFs for the learning process.

In the advanced options you can supply additional data that can improve the analysis. Each of these additional data will result in additional features that will feed the machine-learning.
These data include:

  1. Host proteome archive. Protein FASTA files with the proteome of a known host of the studied bacterium. Multiple files can be included. All these files should be compressed together in a single zip archive.
    This input will be used for homology searches. As effectors interact with host proteins for their function, we expect them to show homology to host proteins.
  2. Archive of proteomes of closely related bacteria without T3SSs. This archive may contain several proteomes records, each in a different FASTA file. These FASTA files should be compressed together in a single zip archive.
    Against each of these proteomes a homology search will be performed. As these bacteria are closely related to the studied bacterium, the vast majority of the proteins in the studied bacterium are expected to have an ortholog in these proteomes. Nevertheless, since they do not encode a T3SS, effectors are not expected to have orthologs in these proteomes. Thus, these features are usually very informative for the machine-learning.
  3. Fully assembled genome question box. By default this box is marked with "no". If the genome you are analyzing is composed of one circular contig, change it to "yes". It will allow calculating features of genome organization.
  4. GFF3 and full genome FASTA files. These input files should be supplied together, and only if the genome is fully assembled (see previous input) and if the bacterium you study is a phytopathogenic bacteria. If this is the case and you supply these data, several known motifs that known TFs of T3Es bind to will be searched in the promoters of the genes.

What is the expected running time of Effectidor?


The running time depends on many factors, among them the input you supply and the load in our servers.
It can take a few minutes and up to several hours. We will send you an email with a link to the results upon submission, if you choose to supply your email address.


For how long will my results be saved in the servers?


The results will be saved in our servers for 3 months.
After 3 months they will be permanently deleted from our servers.


Why do you need my email?


If you give us your email upon submission we will use it to send you a link to your results, and nothing more.
It is an optional input and you do not have to give us your email in order to use Effectidor.


Can I use Effectidor to run the analysis on several genomes simultaneously?


As of today, Effectidor only works for one genome at a time.
You can submit a different job for each genome, but you cannot run a common analysis for more than one genome.
This may change in the future, stay tuned.
