Effectidor allows running a full classification pipline to find uncharacterized type 3 effectors in an input bacterium. It produces both graphical and tabular analysis of the effectors repertoire in the input bacterium. Example output can be found here.















Input specification
The input for the web server is a text file with all the ORFs sequences of the analyzed genome. The file should be in a DNA FASTA format.
You can find instructions to download this file here.
If the analyzed genome is composed of several contigs, or a chromosome and plasmids, the ORFs from the different contigs should be in seperate files, such that each file contains all the ORFs of a specific contig, and the files should be compressed together compressed together in a ZIP archive. Otherwize, a FASTA file can be uploaded.
Input specification
This file should be in a DNA FASTA format. The records supplied in this file must be records from the main ORFs file and they will be treated as known effectors to train the machine-learning algorithm.
This is an optional (and recommended) input. If you choose not to supply it, the records that show homology to our effectors dataset will be used as the known effectors.
Input specification
In this input you may upload host proteome(s). You should upload protein FASTA files, compressed together in a zip file.
If you upload this archive, a BLAST will be ran against all the protein sequences in the fasta files compressed in it. Homology to the host proteins is potentially a very important feature for the classification.
Input specification
In this input you may upload proteomes records of closely related bacteria without a T3SS.
You should upload protein FASTA files, compressed together in a zip file.
Homology to closely related bacteria without a T3SS is a strong negative feature for T3Es classification.
Input specification
If you choose Yes, genome organization features would be extracted.
Do not choose this option if the genome is in several contigs.
Input specification
If the bacterium you are analyzing is a phytopathogenic bacterium and the genome is fully assembled, add a zip archive with full genome sequence FASTA file(s) and a zip archive with corresponding GFF3 file(s). These files will be used for PIP-box, hrp-box, mxiE-box, and exs-box search in the promoters of the ORFs.
These patterns are known recognition motifs of TFs regulating T3Es in phytopathogenic bacteria.

Note that the names of the GFF3 files and the names of the FASTA files should be matching. So if for example in your data there are a chromosome and a plasmid, and the names of the FASTA files are "chromosome.fasta" and "plasmid.fasta", the names of the corresponding GFF3 files should be "chromosome.gff3" and "plasmid.gff3".
The files should not be packed inside directories - directories packed in the zip archive will be ignored. Also files with names starting with a dot (.) or an underscore (_) will be ignored, so make sure not to use these keys at the beginning of your file names.
